## 二、项目概述

### 2.1 项目背景

美团体验与治理组负责全平台用户体验指标的监控与分析，核心指标为"万服"（每万单服务次数）。各业务线每周需要生成体验异动分析报告，用于定位体验问题、指导运营决策。

当前痛点：
- **取数耗时长**：运营/产品需要手动从魔数平台导出数据，耗时1-2小时
- **计算易出错**：数据拿到后需要人工计算YoY、服务量变化占比等指标，容易出错
- **报告撰写难**：分析结论需要人工撰写，格式不统一，质量参差不齐
- **无法定时推送**：周报需要每周手动触发，无法自动定时生成和推送

### 2.2 项目目标

**核心目标**：通过多Skill编排实现"数据查询→智能计算→报告生成→自动推送"的全流程自动化，帮助业务快速定位体验问题，节省运营/产品2-3小时/周。

**量化目标（模拟）**：
- 效率提升：从人工分析2.5-3.5小时 → Agent分析3分钟，效率提升50-70倍
- 使用量：推广阶段目标覆盖3个业务线，每周生成15+份报告
- 满意度：用户满意度评分≥4.0/5.0
- 渗透率：主动使用Agent生成报告的用户占比≥80%

### 2.3 项目范围

### 4.2 功能详细设计

#### F01：自然语言输入（单次分析）

**功能描述**：用户通过自然语言输入业务名称和时间范围，Agent自动执行分析并返回报告。

**输入参数**：
| 参数名 | 参数类型 | 参数说明 | 必填 | 示例 |
|--------|---------|---------|------|------|
| 业务名称 | String | 业务标识，映射数据集ID | 是 | "到餐客服" |
| 时间范围 | String | 本期时间范围 | 是 | "上周"、"2026W12"、"2026-03-16~2026-03-22" |
| 对比方式 | String | 对比期选择 | 否 | 默认"同比"，可选"环比" |

**输出结果**：
| 输出项 | 输出类型 | 输出说明 |
|--------|---------|---------|
| HTML报告链接 | URL | CDN链接，可直接访问 |
| Markdown文本报告 | Text | 7个模块完整内容 |
| 执行日志 | Log | 包含执行时间、耗时、各环节状态 |

**业务规则**：
1. 业务名称映射数据集ID：
   - 到餐客服 → dacan_cs_wanfu_weekly
   - 闪购客服 → shangou_cs_feedback_weekly
   - 企客业务 → qike_cs_inbound_weekly

2. 时间区间计算：
   - "上周" → 自动推算上一完整自然周
   - "W{n}"格式 → 解析为对应自然周
   - 具体日期 → 查询指定日期范围

3. 对比期默认取去年同期（YoY），"环比"时取上一自然周

4. 口径校验：
   - |本期订单量 - 对比期订单量| / 对比期订单量 > 20% → 暂停，提示用户确认

#### F02：创建定时任务

**功能描述**：用户创建定时任务，设置触发时间和业务范围，Agent每周自动生成报告并推送。

**输入参数**：
| 参数名 | 参数类型 | 参数说明 | 必填 | 示例 |
|--------|---------|---------|------|------|
| 触发时间 | Cron | 定时触发时间 | 是 | "每周一9点" → `0 9 * * 1` |
| 业务范围 | String | 分析的业务名称 | 是 | "到餐客服" |
| 分析周期 | String | 分析的时间范围 | 否 | 默认"上一完整自然周" |
| 推送方式 | String | 报告推送方式 | 否 | 默认"邮件推送" |

**输出结果**：
| 输出项 | 输出类型 | 输出说明 |
|--------|---------|---------|
| 任务ID | String | 定时任务唯一标识 |
| 任务配置 | JSON | 包含触发时间、业务范围、推送方式等 |
| 确认信息 | Text | 任务创建确认信息 |

**业务规则**：
1. 触发时间解析：
   - "每周一/周二" → 对应cron表达式
   - "每周一9点" → `0 9 * * 1`
   - "每天" → `0 9 * * *`

2. 分析周期默认取上一完整自然周（cron触发时自动推算）

3. 推送方式默认发送至用户邮箱

4. 创建前需向用户确认参数

#### F03：任务管理

**功能描述**：用户查看/修改/暂停/删除定时任务。

**子功能**：
| 子功能 | 功能描述 | 输入 | 输出 |
|--------|---------|------|------|
| 查看任务 | 列出用户名下所有激活中的定时任务 | "查看我的定时任务" | 任务列表 |
| 修改任务 | 修改任务的触发时间或业务范围 | "把到餐客服周报改到周二9点" | 新的任务配置 |
| 暂停任务 | 保留任务配置，停止触发 | "暂停到餐客服周报" | 任务状态【已暂停】 |
| 恢复任务 | 重新激活任务，恢复触发 | "恢复到餐客服周报" | 任务状态【运行中】 |
| 删除任务 | 永久删除任务 | "删除到餐客服周报" | 任务删除确认 |

**业务规则**：
1. 任务列表格式：
   ```
   当前定时任务列表：
   ① {业务名} 周报 — 每周{X} {Y}点触发 【运行中】 上次执行：{时间}
   ```

2. 修改任务需先确认

3. 删除任务需二次确认

#### F04：HTML可视化报告

**功能描述**：生成HTML可视化报告，上传至S3Plus CDN，返回永久链接。

**报告结构**：
| 区域 | 内容 | 说明 |
|------|------|------|
| 区域1：页头 | 业务名称、分析周期、生成时间 | 大标题红色加粗 |
| 区域2：核心指标卡片区 | 本期万服、对比期万服、YoY、服务量、订单量 | 横向排列4-5个卡片 |
| 区域3：综合分析文字区 | 总结结论、推高主因、压低维度 | 左侧红色竖线装饰 |
| 区域4：分析维度Tab导航 | 城市等级、品类、事件类别、进线渠道、战区、FAQ、明细表 | 每个维度一个Tab |
| 区域5：注意事项模块 | 告警项说明、处置建议 | 黄色警告框 |
| 区域6：明细数据表 | 各维度明细汇总表 | 支持排序、搜索 |

**样式规范**：
| 元素 | 规范 |
|------|------|
| 背景色 | 白色 |
| 主色 | #E8372A（红色，用于上升/重要数据） |
| 辅色 | #4A90D9（蓝色，用于下降） |
| 警告色 | #F5A623（黄色，用于警告框） |

#### F05：Markdown文本报告

**功能描述**：生成Markdown文本报告，包含7个模块完整内容。

**报告模块**：
| 模块 | 内容 | 说明 |
|------|------|------|
| 模块一：核心指标面板 | 本期万服、对比期万服、YoY、服务量、订单量 | 必须输出 |
| 模块二：综合分析 | 总结结论、推高主因、压低维度、总量概览 | 必须输出 |
| 模块三：日万服趋势 | 7日趋势表格、趋势解读 | 必须输出 |
| 模块四：各维度详细分析 | 城市等级、品类、事件类别、进线渠道、战区 | 必须输出 |
| 模块五：各维度明细汇总表 | 跨维度按波动贡献排序 | 必须输出 |
| 模块六：告警解读 | new_category、extreme_value等告警类型 | 必须输出 |
| 模块七：不可用维度标注 | available=false的维度说明 | 条件输出 |

#### F06：邮件推送

**功能描述**：自动发送报告链接到用户邮箱。

**邮件内容**：
| 内容项 | 内容说明 |
|--------|---------|
| 邮件标题 | "{业务名} 体验异动分析周报 - {周次}" |
| 邮件正文 | 报告链接、执行时间、耗时、分析周期 |
| 附件 | 无（报告为CDN链接） |
